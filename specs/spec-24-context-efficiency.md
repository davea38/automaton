# Spec 24: Context Efficiency and Incremental Loading

## Purpose

Every agent invocation re-reads all specs, PRD, AGENTS.md, and IMPLEMENTATION_PLAN.md from scratch. On automaton's own codebase (4146-line main script, 21 specs), each iteration burns enormous input tokens re-reading context. The prompts instruct "use up to 500 parallel Sonnet subagents" for reading specs — massively wasteful for self-build.

## Requirements

### 1. Context Summary File

At each phase transition, generate `.automaton/context_summary.md`: project state summary, completed tasks, remaining tasks, key decisions, recently modified files. Prepended to prompts for the next phase.

### 2. Diff-Based Context for Build Iterations

After the first build iteration, subsequent iterations receive:
- Base `PROMPT_build.md` instructions
- `## Recent Changes` section: `git diff --stat` from last 3 iterations
- `## Current Focus` section: next 5 unchecked tasks from IMPLEMENTATION_PLAN.md
- Replaces "study every file in specs/" instructions

### 3. Self-Build Codebase Overview

When `self_build.enabled`, the build prompt gains a `## Codebase Overview` section generated by the orchestrator: function index of `automaton.sh` (function name, line number, one-line description) extracted via grep.

### 4. Prompt Size Tracking

Track and log prompt size (characters and estimated tokens at 4 chars/token) alongside token tracking for every agent invocation. Makes prompt bloat visible.

### 5. Scaled-Back Subagent Instructions

Replace "up to 500 parallel Sonnet subagents" in `PROMPT_plan.md` and `PROMPT_build.md` with: "Use subagents proportional to codebase size. For a single-file project, 1-3 subagents suffice. Scale up for large multi-file projects."

### 6. Iteration Memory File

After each build iteration, append a one-line summary to `.automaton/iteration_memory.md`: `[BUILD N] Modified file (lines X-Y): description`. Included in context for subsequent iterations.

## Acceptance Criteria

- [ ] `.automaton/context_summary.md` generated at each phase transition
- [ ] Build iterations after the first receive diff-based context
- [ ] Self-build mode generates function index of `automaton.sh` as prompt context
- [ ] Prompt size logged for every agent invocation
- [ ] "500 parallel Sonnet subagents" removed from prompts
- [ ] `.automaton/iteration_memory.md` maintained across build iterations
- [ ] Measured: input tokens per build iteration decrease by at least 30% after optimization
